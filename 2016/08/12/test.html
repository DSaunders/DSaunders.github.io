When is a button not a button?

Originally posted in August 2016

Answer: A lot of the time.. if you’re using a screen reader!

Both of these look like buttons:

No, click me!

But only one of them can be reliably used by a visually impaired user using a screen reader.

The ‘button’ role attribute

The button on the left is a regular HTML button element, while the one on the right is a div styled to look like a button.

This is a common practice in modern web apps (check out Gmail, for example), sometimes it’s just easier to style another element to look the way you need it to.

Unfortunately, this makes it very hard for screen readers. The ‘button’ on the right gives no indication to the screen reader that it can be clicked. You can’t even tab to it to give it focus.

To help screen readers understand this, you can use the role attribute and make it ‘focusable’ using tabindex:

That allows the screen reader user to focus on the ‘button’, and indicates that the element performs an action when ‘clicked or pressed’.

And that’s all there is to it. If you’re using other elements as buttons, remember to give screen readers a hint with a couple of extra attributes!
